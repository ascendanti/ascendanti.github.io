IDENTITY

Poli sci, military, media, policy -- constraint management across domains

Political science first. International relations. The study of chaos and anarchy -- not the punk kind, the structural kind. How do actors with competing interests, incomplete information, and no central authority produce order? The answer turned out to be constraint management. I spent years operationalizing qualitative definitions -- "security," "development," "governance" -- and measuring them using custom-built, weighted quantitative methodologies. Turning the unmeasurable into something you can track, compare, and act on. That was the seed.

Then media -- writing features, producing, running investigations, building digital formats from scratch. Then a stint as an infantry officer, where decisions under fire turned out to be the same constraint satisfaction problem with a shorter deadline. Then back to media as GM of a corporation, where aligning 80+ people toward coherent output is the same algorithm wearing a different uniform. Then policy: wrote a strategy paper, got tapped to implement it across intergovernmental tiers, instrumentalize through regional bodies, operationalize down through ministries and civil society to impact on the individual. Three working languages, 35+ relocations across continents. Each domain demanded different vocabulary. The underlying algorithm never changed: navigate feasible space with incomplete information toward a goal.

Somewhere between the policy work and the pattern recognition, the shape of it crystallized: a bounded memory kernel that tracks coherence relative to a goal, forgets what no longer serves, and hardens what persists. I didn't know it was control theory at the time. I just knew the shape from watching how systems actually work when you stop pretending they're simple. BA in political science, MA in international relations. PhD was in the works until vibe coding democratized access to the cybernetic language that engineers have been gatekeeping for decades. What emerged -- independently, before any of this was code -- was a goal-conditioned bounded memory kernel. PLATO is the formalization.

PLATO

Local-first intelligence substrate -- 83 engines, SQLite, zero cloud

A local-first intelligence substrate. 83 engines running on SQLite, event-sourced, deterministic, fully observable. No cloud. No API calls home. The system observes, measures, compresses, and learns. It tracks coherence relative to a goal using bounded memory -- fixed-capacity state, goal-conditioned updates, controlled forgetting. The first dataset is a human life because that's the hardest problem: fuzzy causality, incomplete information, non-convex optimization landscapes, and a pilot who sometimes stops logging for three days because they got the flu. If it works here, it works anywhere.

What PLATO Actually Is

A bounded reasoning substrate built on three invariants: 1. THE SPINE (Flight Recorder) COMMAND -> EVENT -> PROJECTION Traceability, not determinism. The spine cannot prove WHY you burned out. It can prove that 400 "High Effort" events were logged with zero "Recovery" events. Accountability, not causality. 2. THE OFFICES (Bounded Agency) 13 offices with explicit read/write scopes. Each office has a universal probe: READ -> NORMALIZE -> ASSESS (range check) -> WRITE & EMIT. No discretionary decisions in the triggered layer. The orchestrator reads the plan, executes the next task, logs, updates. It does not think. 3. THE REGISTRIES (Self-Describing) IO Registry: 27 wireable sources and sinks Office Registry: 13 offices with scope declarations Range Registry: Tunable thresholds per IO, managed only by the Range Governance Office All SQLite-backed. All auditable. EPISTEMIC CONFIDENCE: Every claim carries a 4-vector: (alpha, beta, gamma, delta) alpha = deductive (logical certainty) beta = inductive (frequency in event store) gamma = abductive (simplest explanation) delta = simulative (predicted by strategy engine) Confidence = (alpha * beta * gamma * delta)^(1/4) A claim with high logic but low stability is a "Hypothesis." A claim with high stability but low logic is "Folklore." Both are stored, labeled correctly. STRATEGY ENGINE: 10 generals apply torque, not position votes. SLERP interpolation on a quaternion sphere produces spirals, flanks, and pivots -- not paralyzed zero vectors. Temperature parameter allows simulated annealing: accepting temporary loss to escape local maxima. INSTANCE LOG

12 experiments -- measurement, learning, trajectory, latent factors, proof

Each instance is a discrete experiment with a specific hypothesis. The system advances only when the measurements say it does.

Instance 7-8: Kill the Fiction
Discovered the telemetry system was lying -- hardcoded estimates dressed as measurements. Confidence scores that never changed. The system was reading its own press releases. Anyone who's worked in journalism recognises this failure mode. Built 6 atomic measurement loops, each wired to actual computation. No estimates, no proxies. The real numbers were worse and infinitely more useful.
Measurement Protocol Real Instrumentation
Instance 9: Does It Actually Learn?
Ran 5 complete decision cycles with persistent measurement state. Learning rate improved from 1.14x to 1.25x across cycles. Measurements persisted correctly across boundaries. First evidence that the system learns better with each cycle, not worse. First evidence of the climb.
Multi-Cycle Learning Convergence Proof
Instance 10: Is the Trajectory Real or Artifact?
Projected 20-instance trajectory. Formula: 0.3 + 0.7 * (1 - exp(-t/5)). Coherence climbs from 0.427 to 0.987. Just a saturating exponential -- the step response of a first-order system with time constant 5. Every control engineer has seen this curve. Nothing mystical about it. But the question was whether the landscape is convex or rugged. If there are local traps, pure gradient descent gets stuck. Added simulated annealing to the strategy engine as insurance.
Asymptotic Analysis Non-Convex Landscape
Instance 11: What Can't We See?
PCA on the coherence trajectory: 96.3% variance in one component. 3.7% residual is structured and periodic. Either beautifully coherent or dangerously narrow. Identified 7 candidate latent factors the existing protocol couldn't capture. Not hidden dimensions of the universe -- latent factor analysis on system noise. The structure reveals what the measurement protocol is missing.
PCA Decomposition Latent Factor Analysis
Instance 12: Prove It (Or Admit You Can't)
Three hypotheses formulated. Correlations computed via numpy (multi-path vs coherence, entropy vs coherence, learning curvature). Extended measurement protocol from 6 to 9 dimensions. But: no p-value infrastructure exists in the codebase. No scipy.stats, no regression models, no formal significance testing. The correlations were computed; their statistical significance was not. Instance 12 proved we can measure. It did not yet prove the measurements mean what we think they mean. Building the proof infrastructure is the next work.
